# 5-1. 위임장 — 확인·결정 항목

이 양식은 **고객(위임자)**이 **피위임자**에게 선물/포인트 교환 등 업무 처리 권한을 위임하는 **위임장**입니다.  
02-05(시술 전환 확인서)와 유사하게 두 당사자(위임자·피위임자) 정보와 선택/입력 항목이 있습니다. 작업 전 아래 항목에 답변해 주시면 그에 맞춰 입력 화면·PDF·데이터 구조를 설계하겠습니다.

---

## 1. 문서 ID (AL-storecode-YYYYMMDD-000)

- **storecode**: 다른 양식과 동일하게 `storeId` 사용해도 될까요?
- **000** 자리: 데모에서는 001 고정으로 둬도 될까요?

**답변:**
동일하게 사용

**정리:** 문서 ID **AL-{storeId}-YYYYMMDD-001**. storecode = storeId, 데모 001 고정.

---

## 2. 위임자 (본인) 정보

체크리스트: 위임자 성명, 위임자 멤버십 번호. 시스템 표시(읽기 전용): 위임자 회원명.

- **위임자 = 현재 선택한 고객**으로 고정할까요? (02-05의 양도 회원처럼 POS에서 선택한 1명이 위임자)
- **성명**: **실명 우선**(입력 실명 → 없으면 POS 회원명)? 아니면 **회원명만** 읽기 전용 표시?
- **표시 항목**: **멤버십 번호 + 회원명(또는 성명)** 만? 다른 양식처럼 Title, 연락처, 이메일 등 추가할까요?
- **데이터 출처**: 전부 POS에서 주입(선택 고객 기준)?

**답변:**
위임자는 현재 선택된 회원으로 고정.
PDF 양식에서는 실명을 우선으로 주입
입력화면은 다른양식처럼 회원정보를 5가지로 고정해줘.

**정리:** 위임자 = **현재 선택된 회원** 고정. 입력 화면 = 다른 양식과 동일 **5항목**(이름 실명 우선, Title, 회원번호, 연락처, 이메일). PDF = 위임자 성명 **실명 우선**. POS 주입.

---

## 3. 피위임자 정보

체크리스트: 피위임자 성명 (2회 입력), 피위임자 연락처.

- **성명 2회 입력**: **동일한 이름을 두 번** 입력하게 해서 확인용으로 쓸까요? 아니면 **성 + 이름**처럼 나누는 건가요?
- **연락처**: **단일 전화번호** 필드? **국가번호 + 전화번호** (다른 양식처럼)?
- **데이터 출처**: 전부 **직접 입력**? 피위임자도 회원 검색/레이어로 선택할 수 있게 할까요?

**답변:**
위임 받는 사람은 02-05 처럼 선택.
하단 위임자는 선택된 회원이 자동으로 선택되어야함. (PDF 는 실명 우선)

**정리:** 피위임자 = **02-05 수령 회원처럼 레이어 팝업에서 회원 선택**. 위임자(본인) = 현재 선택된 회원 자동 주입(PDF 실명 우선). 피위임자 성명·연락처 = 선택한 회원 정보로 채움.

---

## 4. 위임 유효 기간 (시작일, 종료일)

- **형식**: 체크리스트에 YYYY_MM_DD — **날짜 피커**로 선택하게 할까요? 표시 형식은 **YYYY-MM-DD** / **DD-MM-YYYY** 중 어떤 걸로 할까요?
- **기본값**: 시작일 = 오늘, 종료일 = 오늘+1개월 등 **자동 제안**이 있을까요?
- **검증**: 종료일 ≥ 시작일 등 제한이 있나요?

**답변:**
날짜 피커 (브라우저 기본)는 다른 양식처럼 시스템 형식을 따라가면 될거 같아. (입력화면)
기본값은 둘다 오늘날짜로 채워줘. (하루가 기본). 단, 자동 입력 버튼을 누른경우 7일로 만들어줘
종료일자가 시작일자와 같을수는 있지만 적을수는 없어. 이건 체크해줘.

**정리:** 날짜 피커(브라우저 기본), 다른 양식과 동일 시스템 형식. **기본값**: 시작일·종료일 모두 **오늘**(1일). **자동 생성** 시 **7일** 구간(시작=오늘, 종료=오늘+6 또는 +7). **검증**: 종료일 ≥ 시작일.

---

## 5. 위임 범위 (체크박스)

체크리스트: [Brand] 선물 교환, [Brand] 포인트 사용, 기타 (직접 입력).

- **[Brand]**: 데모에서는 **Sulwhasoo** 등 고정 문자열로 넣을까요? (다른 양식의 brand-name 치환과 동일하게 할까요?)
- **기타**: 선택 시 **직접 입력** 필드 노출? (01-01·04-01의 조건부 입력처럼)
- **선택 방식**: **복수 선택** 가능? 최소 1개 필수?

**답변:**
브랜드 이름으로 치환해줘.
Others는 조건부 입력은 다른 입력처럼 노출해줘.
직접 입력할거라 복수는 고려하지 않아도 괜찮을거 같아. 

**정리:** [Brand] = **브랜드명 치환**(brand-name). **기타** 선택 시 **조건부 직접 입력** 노출(01-01·04-01과 동일). 체크박스 복수 선택 가능, 최소 1개 필수.

## 6. 확인 문구 및 체크박스

체크리스트: "위임자가 위임 범위 내에서 취한 모든 조치를 인정하고 수락하며, 관련 책임을 모두 부담합니다"

- **문구**: **고정 문구**(읽기 전용)로 넣을까요? 데모 **영문**으로 작성해도 될까요?
- **동의 체크박스**: **1개**, **필수** 체크로 할까요? (미체크 시 "다음" 진행 불가)

**답변:**
고정문구로 양식에 있는 문구 그대로 사용해줘.
양식에는 체크박스가 없지만 다른 양식처럼 마지막에 필수로 체크하도록 만들어줘.

**정리:** 확인 문구 = **고정 영문** (원본 양식 문구 그대로). 동의 **체크박스 1개** 하단 추가, **필수** 체크(미체크 시 "다음" 진행 불가).

## 구현 반영 (2026-01-30)

- **입력 폼**: `forms/05-01_form_authorization-letter.html`  
- **PDF**: `forms/pdf/05-01_pdf_authorization-letter.html`  
- **키**: `authorization-letter` (config.js·form-handler·tabs.js 반영)
- **위임자**: 현재 선택 회원 고정, 5항목(이름 실명 우선·Title·회원번호·연락처·이메일). PDF 위임자 성명 실명 우선. **전화번호**: PDF·form-handler에서 **(국가번호) 전화번호** 형식(예: (852) 62341188) 적용.
- **피위임자**: 레이어 "Select Member"로 회원 선택, 성명 2필드·연락처 자동 입력. PDF 피위임자 연락처도 **(국가번호) 전화번호** 형식으로 표시.
- **유효기간**: 시작·종료 날짜 피커, 기본 둘 다 오늘. 자동 생성 시 종료=오늘+7일. 검증: 종료 ≥ 시작.
- **위임 범위**: [Brand] Gift Exchange, [Brand] Points Use, Other + "Please specify" 입력란. **Others 입력란은 01-01처럼 항상 노출**(체크 여부와 무관). 복수 선택 가능, 최소 1개 필수. tabs.js에서 위임 범위 1개 이상 + 동의 체크박스 검증.
- **PDF Scope of Authorization**: 01-01처럼 **체크박스 + Others 입력란** 형태. 선택된 항목에 ✓ 표시.
- **확인 문구**: 고정 영문 + 하단 동의 체크박스 1개 필수.
- **Store/Staff**: 입력 화면에만 표시, PDF 미표시.
- **특이사항 2문구**: 입력·PDF 모두 포함.
