작업 전 아래 항목에 답변해 주시면, 그에 맞춰 입력 화면·PDF·데이터 구조를 설계하겠습니다.

## 1. 문서 ID

03-02(Collagen Drink)에는 문서 ID 형식(CDTNC-storecode-YYYYMMDD-000)이 있습니다. 03-01에는 체크리스트에 문서 ID가 없습니다.

- **03-01에도 문서 ID**를 넣을까요? 넣는다면 **형식**(예: PKTNC-storecode-YYYYMMDD-000)을 정해 주세요.
- 넣지 않는다면 "문서 ID 없음"으로 진행합니다.

**답변:**
아직 고객에게 물어볼 내용이지만 양식에 없는건 무시해도 괜찮아.
나중에 실제 개발할때 보완할 수 있는 내용이야.

## 2. Part 1 — 개인정보 (회원번호, 영문/중문 이름, 연락처, 이메일)

- **데이터 출처**: 전부 **POS에서 주입**(선택 고객 기준) 후 읽기 전용? 일부는 **고객 직접 입력**?
- **이름**: "영문/중문 이름"이 **한 필드**인지, **영문 이름 / 중문 이름 두 필드**로 나눌지?
- **실명 우선**: 다른 양식처럼 **입력화면에서 고객이 입력한 실명**을 이름에 우선 적용할까요?

**답변:**
지금과 동일하게 선택된 고객을 사용하고 실명입력 최우선.

**정리:** 고객정보는 **01-01처럼 5가지 정보**로 표시 — 이름(Name, 실명 우선), Title(호칭), 회원번호(Membership Number), 연락처(Contact: 국가번호 852/853/86 + 전화번호), 이메일(Email). `.customer-info-grid` + `.info-field` 사용, 01-01 Customer Information 구조 동일.

## 3. 구매 전 중요 고지사항 (8개 항목)

- **동의 방식**: 8개 항목에 **항목별 체크박스**를 둘까요, 아니면 **한 개의 "위 내용을 확인하였으며 동의합니다" 체크박스**만 둘까요?
- **스크롤/읽기** *(아래 "스크롤 질문이란?" 참고)*: 동의 체크박스를 **처음부터 바로** 누를 수 있게 할까요? 아니면 **약관을 끝까지 스크롤했을 때**(또는 일정 시간 본 뒤)에만 체크 가능하게 할까요?

**답변:**

가장 마지막에 한번만 하면 됨.
그리고 이미지에 보면 페이지 넘어가기 전에 타이틀이 가장 마지막에 있어. 이건 원본이 좀 잘못된거 같아. 그냥 쭉 이어지는 느낌으로 만들면 될거 같아. 
Important Notice Before Purchase -> 8개 항목
Treatment Packages -> 14개 항목
Product Packages -> 11개 항목

**정리:** 동의 체크박스가 마지막에 있으므로, 체크하려면 어차피 스크롤을 끝까지 내려야 함. 별도 "스크롤 완료 후에만 체크 가능" 로직은 고려하지 않음.

## 4. Part 2 / Part 3 (2·3페이지)

원본이 3페이지 구성인데, 체크리스트에는 Part 1 + "구매 전 중요 고지사항" 8개만 상세히 나와 있습니다.

- **2페이지**: "구매 전 중요 고지사항" 8개만 넣으면 되나요? 아니면 **추가 문단/약관**이 있나요? (원본 -2.png 기준)
- **3페이지**: **서명 블록 + Date/Time**만 넣으면 되나요? 추가로 넣을 문구가 있나요?

**답변:**
위에 답변한것 처럼 트리트먼트 패키지, 제품 패키지가 이미지 하단에 있어. 이건 원본을 내가 잘못 복사한거 같아.
Treatment Packages -> 14개 항목
Product Packages -> 11개 항목
일단 원본에 없는건 따로 생각하지 않아도 괜찮아.

**정리:** 문구는 원본에 있음. **Treatment Packages 14개 항목** = 원본 **2페이지**(-2.png) 내용, **Product Packages 11개 항목** = 원본 **3페이지**(-3.png) 내용. 구현 시 원본 이미지에서 문구 추출하여 사용.
