# 2-6. 시술 연기 확인서 — 확인·결정 항목

작업 전 아래 항목에 답변해 주시면, 그에 맞춰 입력 화면·PDF·데이터 구조를 설계하겠습니다.

---

## 1. 시술 상세 내역 테이블 — 데이터 출처 및 연장일

테이블 열: **시술 항목**, **횟수/가치**, **원래 만료일 (YY/MM/DD)**, **연장일 (YY/MM/DD)** 입니다.

- **테이블 행(연기 대상 시술)**은 어떻게 채우나요?
  - (A) 02-05처럼 POS에서 해당 회원 보유 패키지/시술을 조회한 뒤 **레이어(팝업)에서 선택**
  - (B) 전부 **수동 입력**
  - (C) 그 외 (방식 설명)
- **선택 개수**: 1개 이상만 선택? 2개 이상? 상한 있음?
- **연장일(Extended To)**은 어디서 오나요?
  - 원래 만료일만 POS에서 오고, **연장일은 사용자가 날짜 입력**?
  - **POS에서 "연장 가능일/제안 연장일"**을 내려주고 그중 선택?
  - **원래 만료일 + N일**처럼 시스템이 자동 계산?

**답변:**
패키지 선택은 02-05와 동일하게 처리해주면 됩니다. 
동일하게 1개이상 선택가능 해야 합니다.
오리지널은 POS 데이터 사용, 연장일자는 사용자가 직접 입력할 수 있도록 해야 합니다. (날짜선택 피커 사용)

---

## 2. 매장 위치 (Store Location)

체크리스트에 "매장 위치"가 있고, 원본 1페이지에 Store Location 필드가 있습니다. 02-05에는 Store Registration(매장명)만 있었습니다.

- **Store Location**은 **현재 로그인한 매장명(Store Registration)과 동일**하게 써도 되나요?
- 아니면 **별도 필드**(예: 주소, 층/위치명)로 두어야 하나요?

**답변:**
네네. 양식에 따라 조금 다른거 같은데, 매장명을 사용하면 됩니다.



---

## 3. 고객명(회원명) — 실명 우선

02-05처럼 **입력화면에서 고객이 입력한 실명**을 회원명에 **우선 적용**할까요? (실명 없을 때만 POS 회원명 사용)

**답변:**

이건 모든 양식에 공통적인 내용으로 실명이 최우선.


---

## 4. 문서 ID (TEAL-storecode-YYYYMMDD-000)

- **storecode**: 02-05와 동일하게 `storeId`(예: HK000005) 사용해도 될까요?
- **000** 자리: 데모는 001 고정으로 둬도 될까요?

**답변:**
같은 패턴으로 처리하면 됩니다.



---

## 5. 데모용 "자동 생성" 버튼

02-05처럼 **자동 생성** 버튼을 두어, 고객 정보 + 연기 대상 시술 1~2건(또는 지정 개수) + 연장일 등을 한 번에 채우는 것을 02-06에도 넣을지 여부.

**답변:**
연장일도 자동으로 채우는데, 한달후로 자동으로 선택되게 하면 좋을거 같습니다.

**추가 확인 (자동 생성 시 연장일 기준):** 원래 만료일(Original Expiry) 기준으로 **그날 + 1개월**을 연장일로 채움. (각 행별 적용)




---

## 6. 2페이지 구성

원본 기준 **1페이지**: 제목·고객 정보·시술 테이블·약관, **2페이지**: 문서 ID + 서명 + Date/Time 으로 이해했습니다.

- 2페이지에 서명 블록 외 **추가로 넣어야 할 문구/안내**가 있나요?

**답변:**
원본에 없으면 추가할 필요 없습니다.



---

원본이 English Version인데, 이 양식은 **영문만** 사용할지, **중문/영문 전환**이 필요한지?

**답변:**
향후 중문으로 개발하게 되지만 데모에서는 영문만 사용합니다.
