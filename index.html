업무 프로세스

본 시스템은 아래 프로세스에 따라 단계별로 화면이 구성되어 있습니다. 각 단계는 순차적으로 진행되며, 테스트 데이터를 활용한 시나리오 기반으로 설계되었습니다.

시작하기
안내: 본 화면설계서는 테스트 데이터를 활용한 시나리오 기반 데모입니다. 각 단계는 순차적으로 진행되며, 로그인부터 시작하여 고객 검색 → 상담내용 선택 → 입력/확인 순서로 진행됩니다.
데모 사용 방법
  • 진행 순서: 0단계(로그인) → 1단계(고객검색) → 2단계(양식선택 및 BC 선택) → 3단계(내용입력) → 4단계(최종 확인) → 5단계(전송 완료)
  • 고객검색: 키워드를 입력하면 데모 고객이 필터링됩니다. 고객을 선택해야 다음 단계로 이동할 수 있습니다.
  • 상담내용 선택: 업무항목을 여러 개 선택하면 다음 화면에서 선택한 양식이 탭으로 자동 생성됩니다.
  • 서명: 서명이 필요한 양식이 1개 이상이면 서명 영역이 표시되며, 1회 서명이 모든 해당 양식에 공통 적용됩니다.
  • 초기화: 상단의 “초기화”는 데모 진행 상태(선택한 고객/항목/서명)를 초기 상태로 되돌립니다.
안내: 본 데모는 UX 흐름 이해를 돕기 위한 화면 구성 예시입니다. 실제 운영 로직(POS 연동/검증/저장/발송)은 별도 구현 범위입니다.
